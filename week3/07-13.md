# be-DailyJob 

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 
> 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습: 이미지 업로드를 제외한 게시판 기능에 대한 구현
- (오늘의 목표 중에서) 아직 끝내지 못한 부분: 버튼 비활성화, 에러 페이지, 안내 메시지에 대한 부분

### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: database 메소드를 제네릭 메소드로 변경함.
- 그렇게 판단한 이유: 다운캐스팅을 매번 사용할 때마다 처리해줘야 하는 것이 번거로워서 좋지 않았기 때문임.
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 다운캐스팅이 필요없으므로 사용성에서나 가독성에서나 좋아짐. 다만 여전히 String daoId 같은 파라미터만으로는 어떤 queryprocessor로 처리해야 하는지 알 수 없기 때문에 TargetTable 남아있고 주키 외에도 외래키로 조회하는 경우에 대해서 처리하기 위해서 findkey도 여전히 남아있음.

### 3. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지: html 파일 조작 부분을 깊게 고민하지 않고 생각나는대로 처리한 것.
- 그 접근이 도움이 된 이유: 요청 경로에 따른 처리와 html 파일 조작에 대한 부분에 막혀서 한동안 진행이 안됐었는데 생각나는대로 구현하니 어떻게든 돌아가게는 할 수 있었던 것 같음.
- 다음에도 반복하고 싶은 점: 답답함이 들고 진전이 없을 땐 그냥 생각나는대로 편하게 코딩해서 기능이라도 쳐내는 것.

### 4. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 비효율적이었던 접근/놓치거나 부족했던 점: 데이터베이스 기능, ssr 기능, uri 기능 이런 식으로 나눠서 개발했으면서 각각을 완성했을 때 바로바로 테스트해서 문제가 없는지 확인하지 않은 점.
- 그렇게 된 원인: 각각의 기능을 테스트하는 것도 일이라고 생각해서 그냥 전체를 한번에 다 완성하고 디버깅을 진행했음.
- 다음에는 다르게 해볼 구체적인 행동: 전체를 완성하고 디버깅하는 대신 특정 기능을 완성하는 시점에 디버깅을 하면 전체 디버깅을 할 때는 편하게 할 수 있을 것 같음.
