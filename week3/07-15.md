# be-DailyJob 

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 
> 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습: 이미지 업로드 구현

### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: image 파일 이름을 uuid를 통해 저장
- 그렇게 판단한 이유: 사용자가 지정한 파일 이름을 그대로 가져다 썼을 경우 중복 문제가 있을 수 있음.
- 고려했지만 선택하지 않은 대안: 폴더를 만들어서 관리하는 것

### 3. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 비효율적이었던 접근/놓치거나 부족했던 점: db쪽에서 변경한 사항들이 어디까지 전파될지 생각을 하고 수정할 생각을 했어야 했는데 근시안적으로 수정하고 그 여파를 생각하지 못한 것. 이런 전파를 잡아내기 위한 테스트 코드를 작성하지 않은 것. database 객체로 추상화하고 추상화된 메소드들로만 모든 문제를 해결하겠다고 접근한 것. 트랜잭션을 추가하기 위해서는 이 구조로 불가능한 것을 알면서도 손대지 않은 것. getfilehandler에서 파일 객체를 생성하기 때문에 여기서 파일을 읽고 처리하는 것까지 같이 해야겠다는 생각에 갇혀 있던 것. methodhandler의 역할이 애매하다는 것을 알면서도 그냥 둔 것. 가독성 향상을 위한 작업을 하지 않은 것. DB 모델링을 할 때 usecase 분석을 제대로 하지 않은 것. 
- 그렇게 된 원인: 이 활동에 어떻게 임해야할지에 대해서 명확한 태도를 못 정한 것. 
- 다음에는 다르게 해볼 구체적인 행동: 수정을 하면 테스트가 깨지기 떄문에 추가를 안했지만 테스트는 깨지기 때문에 의미가 있는 것이라는 것을 기억하고 귀찮으면 ai의 도움을 받아서라도 테스트 코드를 추가할 것.
